02Critical Review

What Still Concerns Top-5 Engineers

The foundation is solid. These are next-tier concerns — the ones that surface during Series A due diligence and enterprise security audits.

P0 · SecurityWebSocket Auth is the New Attack Surface
RLS secures your Prisma layer beautifully. But your Express/Socket.io server is a separate process with its own auth model. How does it verify that the user connecting to org-[id] actually belongs to that org? If it trusts a client-supplied org ID, that's a room-hopping vulnerability — any user could subscribe to any org's live activity feed.
FIX: On socket connection, validate a short-lived JWT from Clerk — not a session cookie. The Express server must independently verify the token and extract orgId before allowing room subscription. Never trust client-supplied room names.
P0 · ReliabilityPuppeteer is a Memory Leak Waiting to Happen
Redis queues and concurrency limits help. But Puppeteer browser instances that crash mid-scrape often don't clean up their child processes. Zombie Chrome processes accumulate over days of moderate usage. A single hung page navigation can leak a browser instance indefinitely with no timeout enforcement.
FIX: Wrap every Puppeteer browser in try/finally calling browser.close(). Set a 30s hard navigation timeout per page. Add a cron every 15min that kills Chrome processes over the concurrency limit. Instrument memory to Sentry as a custom metric.
P1 · ComplianceSoft Deletes Need a Retention Policy
Soft deletes preserve data history — but without a retention policy, deleted data accumulates forever. GDPR's right to erasure requires you to fully purge user data on request, which contradicts an infinite soft-delete store. Your DB will also balloon silently over 12-18 months and slow all queries.
FIX: Add an Inngest scheduled job that hard-deletes records where deletedAt < 90 days ago. Add a DELETE /api/user/me endpoint for GDPR erasure requests. Log both to AuditLog with a system actor.
P1 · SecurityThe tRPC Error Boundary May Leak Stack Traces
Forwarding to Sentry is excellent. But ensure the normalized TRPCError returned to the UI does not include internal Prisma error details in production. Prisma errors are verbose — they can expose raw SQL, table names, and column values, which is a serious information disclosure vector in a multi-tenant system.
FIX: In your tRPC error formatter, map all internal errors to generic messages when NODE_ENV === 'production'. Send full context to Sentry server-side only. Never let a raw PrismaClientKnownRequestError reach the browser response body.
P1 · ObservabilitySentry Without Alert Rules Is Just a Filing Cabinet
Sentry captures errors — but do you have alert thresholds that page you? If your error rate triples at 2 AM, do you know within 5 minutes? Without alert rules, you'll review errors weekly after customers complain, not before they do.
FIX: Set Sentry alerts for: spike above 10 new errors/minute, any unhandled rejections from Express, P95 latency above 2s on core tRPC routes. Route to a Slack channel. Takes 20 minutes. Transforms Sentry from passive to active.
P1 · UX QualityStill No Optimistic UI — The Biggest Felt Gap
Even with 100% type-safety and tRPC v11, every mutation still waits for a network round trip before the UI reflects the change. On a 200ms+ connection, checking off a task feels sluggish compared to Linear, which responds instantly. This is the single biggest quality gap between good and great products.
FIX: Use TanStack Query's onMutate to apply changes to local cache immediately, with onError rollback. Start with task completion and deal stage moves. One afternoon of work — transformative in feel.
03Growth Engineering

What Top-5 Engineers Build Next

These aren't bug fixes — they're the features that convert good products into category leaders.

StrategicThe Research Engine Deserves Its Own API Product
You have something genuinely novel: autonomous AI research with real-time terminal streaming that pipes directly into a CRM pipeline. This is a $50-100/month standalone product. Clay.com charges $800/month for less sophisticated enrichment. Your moat is deeper — you just haven't monetized it separately.
OPPORTUNITY: Expose it as a REST API (POST /api/v1/research) with webhook callback on completion. Sell research credits. Build a minimal Node.js SDK. This unlocks a developer market 10x the size of your current target.
RetentionNo Mobile = No Daily Habit Loop
Daily planning, energy tracking, and task completion are highest-frequency interactions — and the ones users want to do from their phone before the day starts. Without mobile, usage will spike Mon-Fri 9-5 and flatline otherwise. Without habit formation, you get churn.
OPPORTUNITY: Add a service worker for offline task caching. Build a mobile-optimized /planning/day layout with large touch targets and swipe-to-complete. Enable Web Push. 70% of mobile gains come from making just this one route excellent.
Demo ImpactCollaborative Cursors Are 80% Already Built
You have WebSocket presence, a canvas, and Socket.io. Collaborative cursors are almost free — broadcast mouse coordinates through the org room and render named cursors on other clients' canvases. Figma's most memorable enterprise sales moment is watching another cursor move in real-time.
OPPORTUNITY: On mousemove in the canvas, throttle to 50ms and emit user position to the org room. Render incoming cursors as floating name labels over the canvas. The quadtree already handles scale — cursors are just another overlay layer.
AI DepthAI Needs Persistent Memory — Right Now It's Stateless
Gemini and Groq are woven throughout — but every call starts from zero context. The AI doesn't know this user has worked in SaaS for 3 years, has a deal closing with Acme Corp, or prefers bullet-point summaries. Without memory, it feels like a generic chatbot. With memory, it feels like a business partner.
OPPORTUNITY: Build a UserAIContext table — industry, role, preferences, recent project summaries, active clients. Inject as a system prompt prefix on every AI call. Two days of work that transforms every AI feature from "impressive" to "indispensable."
